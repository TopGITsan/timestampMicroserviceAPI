Timestamp Microservice

The API endpoint is GET [project_url]/api/timestamp/:date_string?

Date string is valid if can be successfully parsed by new Date(date_string).

Use date strings compliant with ISO-8601 (e.g. "2016-11-20") because this will ensure an UTC timestamp.

Example Usage:

  • [project url]/api/timestamp/2015-12-25
  • [project url]/api/timestamp/1450137600

Example Output:

{"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
